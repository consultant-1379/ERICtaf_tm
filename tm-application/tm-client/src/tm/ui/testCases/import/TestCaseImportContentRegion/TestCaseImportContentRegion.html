When Importing Test Cases

Test case import will update all test cases from the file that have the same test case id. Its important to make sure your test case id's are correct in case a test case is over written accidentally. If a new test case id has been detected it will create a new test case.

Make sure that empty cells in a row contain a blank string otherwise import will fail. The user will need to click on the cell and add a blank space. Excel's backend does not store empty cells in its xml. This causes data to be added into the wrong columns. Test cases will not be imported if a row does not have the correct number of columns.
